## [3.1.0] ‚Äî 2026-04-09

### Added

- **Real-time clock widget** on Tyunnie's Desk ‚Äî displays live hours, minutes, and seconds (`HH:MM:SS`), day of week, and date. Updates every second via `setInterval` in a `useEffect`
- **Weather widget on Desk** (`DeskWeather`) ‚Äî inline component in `Desk.tsx` that pulls from Open-Meteo using the saved `tyunnie_city` from localStorage. Shows temperature, condition label, and weather emoji. Falls back gracefully to "No city set" if no city is configured
- Clock and weather widgets displayed side-by-side in a `grid-cols-2` row below the Tyunnie quote card on the Desk
- **Quick add a task** moved inside the Recent Activity card as a pinned bottom action, separated by a divider line
- Recent Activity capped at 3 items on the Desk for a cleaner, less cluttered layout

### Removed

- `app/demo/` ‚Äî demo route deleted. No longer needed for a personal app
- `app/chat/` ‚Äî standalone chat route deleted. Chat is fully integrated into the dashboard via TyunniePanel
- `app/chat-demo/` ‚Äî chat demo route deleted alongside demo removal

## [1.0.0] ‚Äî 2026-03-20

### Added

Initial release of Tyunnie ‚Äî a full-stack personal productivity assistant web app.

**Core panels:**

- **Calendar** ‚Äî Month, Week, 3-Day, and Year views. Add and delete events. Upcoming events list
- **Tasks** ‚Äî Add tasks with tags (CS, Writing, Personal, Other), due dates, and mark as done
- **Writing** ‚Äî Draft editor with word count, create/edit/delete drafts
- **Projects** ‚Äî Project cards with status (Planning, Active, Paused, Done), progress bars, Gantt chart for date-ranged projects, inline progress slider
- **Snippets** ‚Äî Code editor with line numbers, Tab indentation, Cmd+S to save, live code execution via JDoodle API (Python, JS, TS, Bash), terminal output panel
- **Finance** ‚Äî Monthly income and expense tracking, navigate between months, category tagging, Analytics tab with 6-month bar chart, spending donut, and 50/30/20 rule tracker
- **Music** ‚Äî Upload MP3s to `public/music/`, play/pause/skip/previous, shuffle, repeat modes (none/all/one), seekable progress bar, volume control, album art with spin animation, persistent playback across panel switches

**Tyunnie AI Panel:**

- Context-aware assistant powered by Groq API (Llama 3.3 70B)
- Reads all app data (events, todos, drafts, projects, snippets, finance) and includes it in the system prompt
- Can add events (with confirmation), tasks, drafts, projects, finance entries, and code snippets via natural language
- Action block parsing ‚Äî AI responses include structured `<action>` blocks that are executed client-side after stripping from visible chat
- **Sprite system** ‚Äî 10 named PNG sprites in `public/sprites/` mapped to panels and moods. Mood override with 4-second reset back to panel sprite
- Mini music player embedded in the Tyunnie panel with progress bar and controls
- Greeting bubble on first load from a pool of randomised messages

**Auth:**

- Email/password sign up and login via Supabase Auth
- Google OAuth one-click sign in
- Auto session refresh and redirect to `/auth` on expiry or sign out

**Infrastructure:**

- Next.js 16 (App Router), TypeScript, Tailwind CSS v4
- Supabase PostgreSQL backend with RLS enabled on all tables
- Deployed on Vercel with Vercel Analytics and Speed Insights
- Demo mode at `/demo` ‚Äî no auth required, uses `"demo-user"` guard in `database.ts` to return empty data and suppress Supabase errors
- Fonts: Instrument Serif (`font-serif`), Nunito (`font-sans`), Geist Mono (`font-mono`) via `next/font/google`
- Mobile responsive layout with bottom tab bar navigation
